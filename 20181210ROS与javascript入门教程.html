ROS与javascript入门教程

非ROS端数据交互方法

ROS端目前存在三种接口，一种为调用服务（call service），一种为订阅话题（subscribe a topic），第三种为发布消息（publish message）。其中 调用服务为向服务接口发送请求（request），服务接收到后会返回结果（response）。订阅话题是指ROS端不停地在发布消息（messgae）到相应的话题（topic），非ROS端只要申请订阅这个话题，那么，每次这个话题接收到新的消息时，非ROS端都能够接收到这个消息。发布消息是订阅话题的逆向过程，非ROS端可以直接发布消息（message）到一个话题（topic）上，ROS端订阅这个话题的节点可接收到这个消息并处理。

通俗的来讲，调用服务是主动请求数据，订阅话题是被动推送数据，发布消息是客户端向服务器发消息（比如控制机器人运动）。

1,roslibjs-基本功能

首先加载需要的核心脚本库eventemitter2和roslib :

<script type="text/javascript" src="http://cdn.robotwebtools.org/EventEmitter2/current/eventemitter2.min.js"></script>

0.0

创建Ros节点连接,指定ROS服务器IP地址和端口号

var ros = new ROSLIB.Ros();
// If there is an error on the backend, an 'error' emit will be emitted.
ros.on('error', function(error) {
    console.log('connect error:',error);
});
ros.on('close', function() {
    console.log('Connection closed.');
});
ros.on('connection', function() {
    console.log('Connected to websocket server.');
});
// Create a connection to the rosbridge WebSocket server.
ros.connect('ws://' + IP);

0.1 调用服务

首先，我们要创建一个请求对象（client），其中有三个字段：ros，name和serviceType，ros的值始终是ros，name是指你要请求的服务名字，serviceType是该服务的类型（包括请求的数据格式和返回的数据格式）。

var getParamClient = new ROSLIB.Service({
		ros: ros,
		name: “api/get_params”,
		serviceType: “api_msgs/String”
	});

按照服务的类型定义一个请求变量，本例中，获取参数请求数据格式有一个字段data（ros端定义）

// create a service request
var get_param_req = new ROSLIB.ServiceRequest({
    data: param_name
});

然后就可以发送请求了，返回的结果。

//call service
getParamClient.callService(get_param_req, function(result) {
    console.log("get param", result);
});

0.2 订阅话题

以下以订阅里程计消息为例说明如何在非ROS端订阅话题。

首先，我们要创建一个订阅对象（subscriber），和请求服务一样有三个字段：ros，name和messageType。ros的值始终是ros，name是指你要订阅的话题名字，messageType是该话题消息的类型（消息的数据格式）。

var encSub = new ROSLIB.Topic({
		ros: ros,
		name: “xunjian_nav/encoder”,
		messageType: “xunjian_nav/Encoder”
	});

然后，直接订阅即可，等到有新消息到来时，会自动接收，如果设置了显示，那么就会自动显示在上面。

// subscribe and show
encSub.subscribe(function(msg) {
    $("#leftEncoder").html(msg.leftEncoder);
    $("#rightEncoder").html(msg.rightEncoder);
    $("#vx").html(msg.vx);
    $("#v").html(msg.w);
});

0.3 发布消息

以下以发布机器人速度为例说明如何在非ROS端发布消息到一个特定话题。

首先创建一个发布结构：

cmdVelPub = new ROSLIB.Topic({
		ros: ros,
		name: ‘/remote_cmd_vel’,
		messageType: 'geometry_msgs/Twist'
	});

然后根据消息类型，创建一个消息：

var cmd = new ROSLIB.Message({
		linear: {
			x: 0.4,
			y: 0,
			z: 0
		},
		angular: {
			x: 0,
			y: 0,
			z: 0.6,
		}
	});

最后发布消息：

 cmdVelPub.publish(cmd)
