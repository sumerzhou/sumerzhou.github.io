20170314-VueJS框架之CLI基础

1，安装

在用 Vue.js 构建大型应用时推荐使用 NPM 安装， NPM 能很好地和诸如 Webpack 或 Browserify 模块打包器配合使用。 Vue.js 也提供配套工具来开发单文件组件。

npm:

npm是什么东东？npm其实是Node.js的包管理工具（package manager）。

为啥我们需要一个包管理工具呢？因为我们在Node.js上开发时，会用到很多别人写的JavaScript代码。如果我们要使用别人写的某个包，每次都根据名称搜索一下官方网站，下载代码，解压，再使用，非常繁琐。于是一个集中管理的工具应运而生：大家都把自己开发的模块打包后放到npm官网上，如果要使用，直接通过npm安装就可以直接用，不用管代码存在哪，应该从哪下载。

npm究竟在哪？其实npm已经在Node.js安装的时候顺带装好了。

安装nodeJS：

a,先安装nodeJS。去nodeJS中文官网下载对应平台的安装程序。

b,安装完成后，在电脑菜单栏搜索cmd,在程序上右键以管理员身份打开命令窗口。

c,在命令窗口输入node -v，如果安装正常，你应该看到类似v7.6.0这样的输出。

npm已经在Node.js安装的时候顺带装好了。在命令提示符或者终端输入npm -v，应该看到类似版本的输出。

安装VueJS-CLI:

a,在安装好nodeJS的命令窗口接着输入命令npm install vue 。

b,安装完后，输入命令 vue 。

c,在报“不是内部或外部命令……”错误后，去百度搜索npm淘宝镜像网站，使用定制的 cnpm命令行工具代替默认的 npm。

d,对于再次输入vue仍然报错的情况，使用global全局命令：npm install vue –global。

e,待跳出Vue@（版本号）之后，输入npm install –global vue-cli安装CLI。

    //全局安装 vue-cli
	$ npm install --global vue-cli
	// 创建一个基于 webpack 模板的新项目
	$ vue init webpack my-project
	//安装依赖，走你
	$ cd my-project
	$ npm install
	$ npm run dev

f,安装成功后，输入vue,待出现usage，commands,options等信息，说明安装成功。

g,webpack初始化（vue init webpack my-project命令），设置项目文件夹.。之后每次新建新项目文件夹，从此命令开始。

h,cd到新创建的文件夹（cd my-project）,配置文件名及其他，默认设置的话一直回车。

i,npm install(如果安装失败报错，使用cnpm install命令代替)。

j,最后输入npm run dev命令，完成安装。

2,打包APP

Hbuilder打包：

首先，在要打包的文件夹上右键选择‘转换成移动APP’，配置基本信息。

APP云端打包：

a,下载Java runtime，用以android证书的生成。

b,到安装好Java的文件夹，进入到bin文件，打开命令（shift+右键），输入命令(keytool -genkey -alias yourapp -keyalg RSA -validity 20000 -keystore yourapp.keystore)，命名好自己的证书文件名。

c,复制生成的秘钥文件到要打包的文件内。

若在电脑端模拟安卓手机，可安装模拟器：夜神安卓模拟器。

Hbuilder上的APP框架：

MUI,最接近原生APP体验的高性能前端框架。

其他打包工具：

appcan,ionic,android studio/android SDK。

其余实用工具：

构建工具grunt, 聚合数据API，百度API, 阿凡达数据API,安卓手机通用驱动。

3,路由

要使用vuejs的路由功能，首先在头部引入vue-router.js文件。

    //1,定义组件（路由）
    const home={template:'#home'};
    const about={template:'#about'} ;
    //2,定义路由
    //每个路由应该映射一个组件。其中‘component’可以是
    //通过Vue.extend()创建的组件构造器，
    //或者，只是一个组件配置对象
    const  routes=[
        {path:'/home',component:home},
        {path:'/about',component:about}
    ];

    //3,创建router实例，然后传‘routes’配置
    const router=new VueRouter({
        routes    //缩写，相当于routes：routes
    });

    //4,创建和挂载根实例
    //记得要通过router配置参数注入路由，从而让整个应用都有路由功能
    const app=new Vue({
        router
    }).$mount('#app');
    //现在，应用已经启动了！
    	<div id="app">
        <ul class="list-group">
            <router-link to="/home" class="list-group-item">Go to Foo</router-link>
            <router-link to="/about" class="list-group-item">Go to Bar</router-link>
        </ul>
	    <router-view></router-view>
	</div>

4,AMD和CMD

AMD和CMD最大的区别是对依赖模块的执行时机处理不同，注意不是加载的时机或者方式不同。AMD推崇依赖前置，在定义模块的时候就要声明其依赖的模块;CMD推崇就近依赖，只有在用到某个模块的时候再去require。

很多人说AMD用户体验好，因为没有延迟，依赖模块提前执行了，CMD性能好，因为只有用户需要的时候才执行。
