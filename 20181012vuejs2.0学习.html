vuejs 2.0 学习

1,vue简介

vue同样采取了虚拟DOM形式。

Vue.js 使用了基于 HTML 的模板语法，允许开发者声明式地将 DOM 绑定至底层 Vue 实例的数据。在底层的实现上，Vue 将模板编译成虚拟 DOM 渲染函数。

"Mustache"语法 (双大括号)只能存在单行语句，且不能作用在HTML属性中。

2,Vue基础要点

>> 事件处理：

事件修饰符:

stop,prevent,once,self

按键修饰符:

enter,tab,delete,esc,space,down,up

>> 指令：

v-model修饰符：

lazy，number，trim

>> 组件：

在ul、table、select这样的元素里允许包含的元素有限制，在自定义组件中使用这些受限元素时会导致一些问题，变通的方案是使用特殊的is特性。

单文件组件，由三个部分组成：Style，Template，Script

style :

scoped:样式只在当前组件内生效

props:

子父组件交互通信：

父 -> 子：props

子 -> 父：emit Event

数据传递类型限制/验证;

子组件通过自定义事件给父组件通信:

this.$emit(key,value)

使用插槽分发内容:

使用slot插槽使得子父集的传递不仅是数据，还可以是视图。

动态组件,keep-alive:

通过使用保留的component元素，动态地绑定到它的is特性，让多个组件可以使用同一个挂载点，并动态切换。

实践中，不用component元素和is特性，直接用双括号放置组件的动态变量，也可以实现。

keep-alive可以把切换出去的组件保留在内存中，它可以保留组件的状态或避免重新渲染。

>> 过渡与动画:

可以有css和js两种方式实现。建议使用css，因为js对性能消耗要大。

· 在css过渡和动画中自动应用class。

· 可以配合使用第三方css动画库，如Animate.css。

>> 自定义指令:

有全局指令和局部指令之分。

>> 过滤器

过滤器可以在两个地方使用：双花括号插值和v-bind表达式（后者从2.1.0+开始支持）。加载javascript表达式尾部，由“管道”符号指示。

>> 跨域解决方案：

开发时的跨域用jsonp一般是不适合的。像vue这样前后端分离开发，上线后程序的客户端代码和服务器代码是放在一起的，不存在跨越的问题的。

vue给我们提供了一个配置方案。在config里面有个index.js，里面有个专门的解决跨域的方案proxyTable。例如:

// 第一步  /config/index.js
module.exports = {
  dev: {
    ...
    proxyTable: {
      "/api":{
        target:"http://localhost:3000",
        changeOrigin:true,
        pathRewrite:{
          '^/api':''
        }
      }
    },
    host: 'localhost', // can be overwritten by process.env.HOST
    port: 8080, // can be overwritten by process.env.PORT, if port is in use, a free one will be determined
    ...
// 第二步  /main.js
Vue.prototype.HOST = '/api'
    // 读HOST相当于读了config/index里proxyTable的‘api’的target地址

然后一个url地址就可以用this.HOST+path来表示了。

注意：此种跨域解决方案，只能适用于测试阶段，对上线后的跨域不行。因为打包的时候，不会具备服务器，不能跨域了。

>> Mock数据模拟:

1, 自己创建json文件。使用get请求形式访问数据

优点：方便，快捷

缺点：只能存在get请求

2，项目中集成服务器（比如node环境），模拟各种接口

优点：模拟真实线上环境

缺点：增加开发成本

3，直接使用线上数据

优点：真实

缺点：不一定每个项目都存在这样的数据

>> 路由(vue-router 2)

vue-router@2.x只适用于vue 2.x。

安装：npm install vue-router

通过Vue.use()明确地安装路由功能：

// main.js
    import Vue from 'vue'
    import VueRouter from 'vue-router'
    Vue.use(VueRouter)
    const router = new VueRouter({ routes:[{path:'/',component:HellowWorld} ] })
    new Vue({ router }).$mount('#app')

<router-view>组件是一个功能组件，渲染路径匹配到的视图组件。它渲染的组件还可以内嵌自己的<router-view>，根据嵌套路径，渲染嵌套组件。

<router-link>组件通过to属性指定目标地址，实现点击导航。默认渲染成带有正确连接的a标签。

路由嵌套：new VueRouter（）时候，在routes数组下对应有嵌套需求的对象中增加children属性.

路由传递参数:
    //方式一： 带查询参数，下面的结果为/user?count=123
    <router-link :to="{ path:'/user', query:{count:123}}">Register</router-link>
    //方式二： 下面的结果为/user/123
    <router-link :to="{ name:'user', params:{ count:123 }}">User</router-link>

对应的路由里面这样写：

// router/index.js
routes: [
    {
      path: '/user/:count',
      name: 'user',
      component: User,
    },
],
获取路由传递的参数：

方式一的路由参数传递方式，可以通过$route.query.count这样的方式，来获取到传递的值。

方式二的路由参数传递方式，可以通过$route.params.count这样的方式，来获取到传递的值。

选中路由的高亮效果：

a,设置默认的.router-link-active类的样式；

b,默认值可以通过路由的构造选项linkActiveClass来全局配置。

vuex是vue应用程序的状态管理模式，通俗讲就是使程序组件共享数据。

>> vue-resource

Vue.js是数据驱动的，这使得我们并不需要直接操作DOM，如果我们不需要使用jQuery的DOM选择器，就没有必要引入jQuery。vue-resource是Vue.js的一款插件，它可以通过XMLHttpRequest或JSONP发起请求并处理响应。也就是说，$.ajax能做的事情，vue-resource插件一样也能做到，而且vue-resource的API更为简洁。另外，vue-resource还提供了非常有用的inteceptor功能，使用inteceptor可以在请求前和请求后附加一些行为，比如使用inteceptor在ajax请求时显示loading界面。

和Vue.js一样，vue-resource除了不支持IE 9以下的浏览器，其他主流的浏览器都支持。

vue更新到2.0之后，作者就不在对vue-resource更新了，而是推荐axios。

4,简易实例项目演示

环境要求： Node.js (>=6.x, 8.x preferred), npm version 3+

安装 vue-cli :

npm install -g vue-cli

初始化项目(我的项目名vue2.0dev)：

vue init webpack vue2.0dev

初始化完成后，项目的环境就搭建好了，包括eslint、babel、webpack、router等，在初始化过程中可以根据命令行窗口的提示安装。

项目目录结构：

.postcssrc.js：postcss配置文件

static：静态文件目录。服务器启动之后，可以直接访问到它下面的内容。通常会把一些json文件、文档放在static下面。

config文件夹：一般是针对开发上服务器的配置。比如开发环境的配置、生产环境配置、针对服务器跨域的一些配置、host等。

build文件夹:包含服务器和webpack配置。

main.js：入口文件。

App.vue:第一个vue组件。

assets文件夹：资源目录，比方公共的css、字体、图片等。

项目过程:

安装axios：npm install axios --save

引入加载axios：

// main.js
    import Axios from "axios"
    Vue.prototype.$axios = Axios;
    // 在Vue原型上创建了一个$axios 对象。
