angular4基础-项目实践(Angular打造企业级协作平台)

前言:AngularJS、 Angular 2、Angular 4

常说的 Angular 1 是指 AngularJS； 从Angular 2 开始已经改名了。不再带有JS，只是单纯的 Angular。

相比 Angular 1.x， Angular 2 的体积更小，快。 如果仅仅用于PC 端的WEB开发， Angular 1.x足以应对， 如果是用于 mobile app ，在用户体验方面，略显捉襟见肘！推出 Angular 2 是迫于 mobile apps 的需要。Angular 1写的代码无法用在Angular 2上,Angular 1.x 是基于JavaScript的框架，而Angular 2是基于TypeScript的框架。

Angular 4 是 Angular 2 的升级版本，Angular 4 比 Angular 2 更快。

1,环境搭建

视频教程使用的软件版本：angular4.x, @angular/cli-1.1.0, node6.9.2

>> 系统基础环境：

node：安装node是为了使用npm管理项目依赖的软件包.

@angular/cli是angular团队推出的一个脚手架工具，用于快速构建angular应用。使用angular CLI使我们无须理会复杂的配置，更专注angular。

chrome安装Augury插件，在网址chrome-extension-downloader.com，页面输入id（elgalmkoelokbchhkhacckoklkejnhcd）后下载。

>> angular CLI的几个常用命令：

ng --version:查看安装的angular的版本号

ng new 项目名：新建angular项目

ng build -prod:生产环境编译。如果没有加prod，生成出的并不是优化的代码，加了之后会有代码压缩等。

ng serve:启动发开服务器

>> Mock Rest API:

json-server :用于快速搭建REST API的利器。利用json文件快速生产非常接近于真实的rest API。

安装：npm install -g json-server

使用：在项目中构造一个json文件用来存放API数据，然后用“json-server 文件路径”启动API host服务。这时在浏览器中访问localhost:3000可以看到json里的接口和数据。支持get、post、put、patch、delete等REST命令操作。

>> 测试REST API：

上面那样是使用浏览器测试get请求，但浏览器并不能测试所有的REST API，比如post、put。

几种常见的API测试工具：

a, 使用postman测试常用的API

b, 使用VSCode的REST Client插件

>> 常见问题解决：

安装某个包出现错误,可以适当的清除缓存试试。删掉重新安装：

先npm uninstall [package]

然后清除缓存npm cache clean --force

再重新安装刚刚安装出错被删除的package。

某些情况下可以删除整个node_modules 模块：rm -rf node_modules

在src/app下添加模块:

ng g m moduleName

用上面的命令在app文件夹下增加core和shared模块。core模块在系统中只加载一次。share模块作为共享模块，最主要的作用是把公用的东西导入并导出，以后有其他模块需要这里导入的模块的话，只要导入shared模块就可以了，因为分别在每一个模块里去导入导出的话会很繁琐。还有一些共享的组件可以放在shared模块中。

// core/core.module.ts
import { NgModule, SkipSelf, Optional } from '@angular/core';
import { CommonModule } from '@angular/common';

@NgModule({
  imports: [
    CommonModule
  ],
  declarations: []
})
export class CoreModule {
  constructor(@Optional() @SkipSelf() parent: CoreModule) {
    //SkipSelf注解，让函数不要在自己这里打转，去父级那里寻找
    //Optional说明这个依赖是可选的
    if (parent) {
      throw new Error('模块已经存在，不能再次加载！');
    }
  }
}
// shared/shared.module.ts
import { NgModule } from '@angular/core';
import { CommonModule } from '@angular/common';

@NgModule({
  imports: [
    CommonModule
  ],
  exports: [
    CommonModule
  ],
  declarations: []
})
export class SharedModule { }

在app.module.ts中把core模块导入,这样只需在根模块中加载一次core模块，其他时候就不加载了：

// app.module.ts
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { CoreModule } from './core/core.module';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule,
    CoreModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

MdIcon导入svg图片导入当成图标使用：

在文件夹assets中放入一张svg格式的图片（shouce.svg）。为了避免在需求svg资源组件内哪使用哪加载导致的图标资源加载重复和分散问题，推荐在app目录下创建一个工具文件夹utils:

// app/utils/svg.util.ts
import {MdIconRegistry} from '@angular/material';
import {DomSanitizer} from '@angular/platform-browser';

export const loadSvgResources = (ir: MdIconRegistry, ds: DomSanitizer) => {
  ir.addSvgIcon('gifts', ds.bypassSecurityTrustResourceUrl('assets/shouce.svg'));
} //DomSanitizer构建安全的url

这时候，如果去像header之类组件里的constructor函数调用工具类，还是存在重复调用的问题，每个用到svg转的icon图标时都会调用。最好的入口（本项目中）是在core.module.ts中，导入一次，其他组件都不用关心这一状态，只要用就可以了。加载这件事在core module初始化中去做。

material icon如果要导svg是需要http依赖的，所以在core.module.ts中导入HttpModule:

// core.module.ts
import { HttpModule } from '@angular/http';
import {MdIconRegistry} from '@angular/material';
import {DomSanitizer} from '@angular/platform-browser';
import { loadSvgResources } from '../utils/svg.util'
...
@NgModule({
  imports: [
    ...
    HttpModule
  ],
  ...
})
export class CoreModule {
  constructor(
    @Optional() @SkipSelf() parent: CoreModule,
    ir: MdIconRegistry,
    ds: DomSanitizer
    ) {
    ...
    loadSvgResources(ir, ds);
  }
}

对话框dialog的使用:

对话框很特殊（menu也是），需要在模块中的entryComponents中声明。因为他们都是事先需要隐藏的。一开始模块加载时看不到，在某个操作后出现，对于这种情况需要用到预加载。

传递数据也比较特殊：从调用者发送。

接收数据：在dialog的构造函数里面去注入MD_DIALOG_DATA令牌，就能取得调用者传递来的数据。constructor(@Inject(MD_DIALOG_DATA) public data:any){}

构建confirm-dialog组件：

// confirm-dialog.component.ts
import {Component, Inject, OnInit} from '@angular/core';
import {MD_DIALOG_DATA, MdDialogRef} from '@angular/material';
@Component({
  selector: 'app-confirm-dialog',
  template: `
    <h3 md-dialog-title>{{title}}</h3>
    <div md-dialog-content>
     {{content}}
    </div>
    <div md-dialog-actions>
      <button type="button" md-raised-button color="primary" (click)="onClick(true)">确定</button>
      <button type="button" md-button md-dialog-close (click)="onClick(false)"> 取消</button>
    </div>
  `,
  styles: []
})
export class ConfirmDialogComponent implements OnInit {
  title = '';
  content = '';
  constructor(
    private dialogRef: MdDialogRef<ConfirmDialogComponent>,
    @Inject(MD_DIALOG_DATA) private data) { }
  ngOnInit() {
    this.title = this.data.title;
    this.content = this.data.content;
  }
  onClick(result: boolean) {
    this.dialogRef.close(result);
  }
}
